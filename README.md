# micromech

Lightweight [OLAS](https://olas.network/) mech runtime. Create, stake, and run autonomous mechs that answer on-chain requests using pluggable tools — from local LLMs to API calls.

Replaces the 100K+ line Open Autonomy stack with ~2K lines of async Python.

## Why micromech?

The existing [mech-server](https://github.com/valory-xyz/mech-server) requires Open Autonomy, Tendermint BFT consensus, and dozens of framework dependencies. micromech strips all of that away:

| | mech-server | micromech |
|---|---|---|
| Runtime | Open Autonomy + Tendermint (4 agents) | Single async process |
| Tool loading | `exec()` from IPFS | `importlib` (safe) |
| Concurrency | Sequential (`max_workers=1`) | Concurrent (configurable pool) |
| Persistence | None (re-scans blocks on restart) | SQLite (zero lost requests) |
| Dependencies | ~50 packages | ~10 core packages |
| Lines of code | ~100K | ~2K |

## Features

- **Persistent request queue** — SQLite with WAL mode. Requests survive crashes and restarts.
- **Concurrent tool execution** — asyncio with configurable concurrency limit and per-tool timeouts.
- **Pluggable tools** — built-in echo + local LLM (Qwen 0.5B on CPU), plus Valory-compatible tool loader.
- **On-chain listener** — polls MechMarketplace for Request events, filters by your mech address.
- **HTTP API** — accept off-chain requests via `POST /request`, monitor via `GET /status`.
- **Web dashboard** — real-time queue stats, recent requests, tool list.
- **CLI** — manage everything from the terminal.
- **iwa integration** — optional dependency for wallet, RPC rotation, Safe transactions.

## Quick Start

```bash
# Install
pip install micromech[cli,web]

# Initialize config
micromech init

# List available tools
micromech tools

# Test the echo tool
micromech test-tool echo "Will ETH hit 10k by 2027?"

# Run the server (HTTP + event listener + delivery)
micromech run
```

## Installation

```bash
# Core only (models, persistence, tools)
pip install micromech

# With CLI
pip install micromech[cli]

# With web dashboard
pip install micromech[web]

# With local LLM tool (Qwen 0.5B, CPU-friendly)
pip install micromech[llm]

# With on-chain operations (requires iwa)
pip install micromech[chain]

# Everything
pip install micromech[cli,web,llm,chain]
```

Requires Python 3.12+.

## Architecture

```
Request in                                        Response out
    |                                                  |
    v                                                  v
[On-chain events] --> [Persistent Queue] --> [Tool Executor] --> [Delivery Manager]
[HTTP endpoint]   -->    (SQLite)            (async pool)        (on-chain tx)
```

### Modules

```
src/micromech/
  core/             # Config (Pydantic), models, SQLite queue, errors
  tools/            # Tool ABC, registry, echo, LLM, Valory compat
  runtime/          # Server, executor, event listener, delivery, HTTP
  integration/      # iwa bridge (optional)
  web/              # Dashboard (FastAPI + Jinja2)
  cli.py            # Typer CLI
```

### Request Lifecycle

1. **Ingestion** — request arrives via on-chain event or HTTP POST
2. **Persistence** — immediately written to SQLite (`status=pending`)
3. **Dedup** — duplicate request IDs are rejected
4. **Execution** — tool runs with timeout enforcement (`status=executing` → `executed`)
5. **Delivery** — result submitted on-chain via `deliverToMarketplace()` (`status=delivered`)
6. **Recovery** — on restart, pending/executing requests are re-queued automatically

## Configuration

Config lives at `~/.micromech/config.yaml`. Generated by `micromech init`.

```yaml
runtime:
  host: 0.0.0.0
  port: 8000
  log_level: INFO
  max_concurrent: 10          # concurrent tool executions
  request_timeout: 300        # per-request timeout (seconds)
  event_poll_interval: 15     # on-chain polling interval (seconds)
  delivery_batch_size: 10     # responses per delivery cycle
  delivery_interval: 10       # delivery cycle interval (seconds)

mech:
  chain: gnosis
  mech_address: null          # your mech contract address
  marketplace_address: "0x735FAAb1c4Ec41128c367AFb5c3baC73509f70bB"

persistence:
  db_path: ~/.micromech/data/micromech.db

llm:
  model_repo: Qwen/Qwen2.5-0.5B-Instruct-GGUF
  model_file: qwen2.5-0.5b-instruct-q4_k_m.gguf
  max_tokens: 256
  context_size: 2048

tools:
  - id: echo
    enabled: true
```

## Tools

### Built-in

| Tool | Description | Timeout | Dependencies |
|------|-------------|---------|-------------|
| `echo` | Returns the prompt as-is | 5s | None |
| `llm` | Local LLM (Qwen 0.5B Q4, CPU) | 120s | `micromech[llm]` |

### Custom Tools

Create a tool by subclassing `Tool`:

```python
from micromech.tools.base import Tool, ToolMetadata

class MyTool(Tool):
    metadata = ToolMetadata(
        id="my-tool",
        name="My Tool",
        description="Does something useful",
        timeout=30,
    )

    async def execute(self, prompt: str, **kwargs) -> str:
        # Your logic here
        return '{"result": "answer"}'
```

Register it:

```python
from micromech.tools.registry import ToolRegistry

registry = ToolRegistry()
registry.load_builtins()
registry.register(MyTool())
```

### Valory-compatible Tools

micromech can load tools in the [Valory mech format](https://github.com/valory-xyz/mech):

```
my_tool/
  component.yaml    # name, entry_point, callable, version
  my_tool.py        # Python module with run(**kwargs) function
```

```python
from micromech.tools.valory_compat import load_valory_tool

tool = load_valory_tool(Path("my_tool/"))
registry.register(tool)
```

Uses `importlib` (not `exec()`) for safe module loading.

## On-chain Integration

micromech uses [iwa](https://github.com/dvilelaf/iwa) for blockchain operations:

- **Wallet** — key management, transaction signing
- **Chain interfaces** — RPC auto-rotation, rate limiting
- **Gnosis Safe** — multisig transaction execution
- **Contracts** — MechMarketplace, staking, service registry

```bash
pip install micromech[chain]
```

The server automatically detects and uses iwa when installed. Without iwa, micromech runs in HTTP-only mode (no on-chain listening or delivery).

## Valory Compatibility

micromech is compatible with the OLAS mech ecosystem:

- **Request format**: `{"prompt": "...", "tool": "..."}`
- **Response format**: `{"requestId": N, "result": "...", "tool": "..."}`
- **Tool interface**: `run(**kwargs) -> (result, prompt, context, artifact, callback)`
- **On-chain events**: `Request(address indexed mech, bytes32 requestId, bytes data)`
- **Delivery**: `deliverToMarketplace(requestIds[], datas[])`
